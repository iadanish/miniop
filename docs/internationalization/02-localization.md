# Localization Guide

## Overview

Localization (l10n) extends internationalization to adapt MiniOp's content, UI, and workflows for specific regions. This covers video caption localization, asset management, locale-specific content rules, and translation pipeline integration for both the free tier and scaled production.

## Locale-Specific Content Rules

### Date and Time

Video timestamps display differently per locale. MiniOp stores all timestamps in ISO 8601 (UTC) and converts at render time:

```ts
// src/utils/datetime.ts
import { useIntl } from 'react-intl';

export function useLocalizedTimestamp(isoString: string) {
  const intl = useIntl();
  return {
    absolute: intl.formatDate(isoString, {
      dateStyle: 'medium',
      timeStyle: 'short',
    }),
    relative: intl.formatRelativeTime(
      Math.floor((Date.now() - new Date(isoString).getTime()) / -1000),
      'second'
    ),
    duration: (seconds: number) =>
      intl.formatDuration(seconds, {
        format: seconds >= 3600 ? ['hours', 'minutes', 'seconds'] : ['minutes', 'seconds'],
      }),
  };
}
```

### Text Direction and Layout

See `03-rtl-support.md` for full RTL handling. Key localization rule: never hardcode `margin-left` or `padding-right`. use logical properties:

```css
/* Wrong */
.clip-card { margin-left: 16px; text-align: left; }

/* Correct */
.clip-card { margin-inline-start: 16px; text-align: start; }
```

### Number Formatting by Region

Clip duration, view counts, and export sizes must respect regional conventions:

```tsx
// src/components/ClipMeta.tsx
import { useIntl } from 'react-intl';

export function ClipMeta({ views, duration, fileSize }: ClipMetaProps) {
  const intl = useIntl();

  return (
    <div className="clip-meta">
      <span>{intl.formatNumber(views, { notation: 'compact', maximumSignificantDigits: 2 })}</span>
      {/* en: "1.2M" | ja: "120万" | de: "1,2 Mio." */}
      <span>{intl.formatNumber(fileSize, { style: 'unit', unit: 'megabyte' })}</span>
      {/* en: "256 MB" | fr: "256 Mo" */}
    </div>
  );
}
```

## Video Caption Localization

MiniOp generates AI captions for video clips. These captions must be localizable.

### Caption File Format

Export captions as WebVTT with locale metadata:

```
WEBVTT
Kind: captions
Language: en

00:00:01.000 --> 00:00:04.000
This is a sample caption

00:00:05.000 --> 00:00:08.000
Generated by MiniOp AI
```

### Caption Translation Pipeline

```ts
// src/services/captionTranslate.ts
interface CaptionSegment {
  start: number;
  end: number;
  text: string;
}

export async function translateCaptions(
  segments: CaptionSegment[],
  sourceLocale: string,
  targetLocale: string
): Promise<CaptionSegment[]> {
  const endpoint = import.meta.env.VITE_TRANSLATION_API;

  const resp = await fetch(`${endpoint}/translate`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      texts: segments.map((s) => s.text),
      source: sourceLocale,
      target: targetLocale,
      preserve_timestamps: true,
    }),
  });

  const { translations } = await resp.json();

  return segments.map((seg, i) => ({
    ...seg,
    text: translations[i],
  }));
}
```

### Free Tier: Manual Caption Editing

Free tier users manually edit translated captions via a simple textarea. No API integration:

```tsx
export function CaptionEditor({ segments, locale }: CaptionEditorProps) {
  const intl = useIntl();
  const [edits, setEdits] = useState(segments);

  return (
    <div>
      {edits.map((seg, i) => (
        <div key={i} className="caption-row">
          <span className="timestamp">
            {intl.formatNumber(seg.start, { style: 'unit', unit: 'second' })}
          </span>
          <textarea
            value={seg.text}
            onChange={(e) => {
              const next = [...edits];
              next[i] = { ...seg, text: e.target.value };
              setEdits(next);
            }}
            dir={isRTL(locale) ? 'rtl' : 'ltr'}
            lang={locale}
          />
        </div>
      ))}
    </div>
  );
}
```

### Scaled Production: AI Translation Service

For production, integrate with DeepL or Google Cloud Translation:

```ts
// src/services/translationProvider.ts
import DeeplClient from 'deepl-node';

const translator = new DeeplClient(process.env.DEEPL_API_KEY);

export async function batchTranslate(
  texts: string[],
  targetLang: string,
  sourceLang?: string
): Promise<string[]> {
  const results = await translator.translateText(
    texts,
    sourceLang || null,
    targetLang,
    {
      preserveFormatting: true,
      formality: 'default',
      splitSentences: 'nonewlines',
    }
  );
  return results.map((r) => r.text);
}
```

Configuration for multiple providers:

```ts
// src/config/translation.ts
export const translationConfig = {
  provider: import.meta.env.VITE_TRANSLATION_PROVIDER || 'manual',
  deepl: {
    apiKey: import.meta.env.DEEPL_API_KEY,
    endpoint: import.meta.env.DEEPL_ENDPOINT || 'https://api-free.deepl.com',
    usageLimit: 500000, // characters per month on free plan
  },
  google: {
    projectId: import.meta.env.GOOGLE_PROJECT_ID,
    location: 'global',
    glossaryId: import.meta.env.GOOGLE_GLOSSARY_ID,
  },
  fallback: 'manual',
};
```

## Asset Localization

### Locale-Specific Images

Some images (screenshots, marketing assets) need per-locale versions:

```
public/
  images/
    features/
      en/
        dashboard.png
        export-flow.png
      ar/
        dashboard.png
        export-flow.png
      ja/
        dashboard.png
```

Reference with the locale hook:

```tsx
import { useLocale } from '../hooks/useLocale';

export function FeatureScreenshot({ name }: { name: string }) {
  const { locale } = useLocale();

  return (
    <picture>
      <source srcSet={`/images/features/${locale}/${name}.png`} />
      <img src={`/images/features/en/${name}.png`} alt={name} />
    </picture>
  );
}
```

### Font Loading

Different locales require different font stacks:

```ts
// src/config/fonts.ts
export const fontStacks: Record<string, string> = {
  en: '"Inter", "Segoe UI", system-ui, sans-serif',
  ar: '"IBM Plex Sans Arabic", "Segoe UI", system-ui, sans-serif',
  ja: '"Noto Sans JP", "Hiragino Sans", "Yu Gothic", system-ui, sans-serif',
  zh: '"Noto Sans SC", "PingFang SC", "Microsoft YaHei", system-ui, sans-serif',
  ko: '"Noto Sans KR", "Malgun Gothic", system-ui, sans-serif',
};

export function getFontStack(locale: string): string {
  const lang = locale.split('-')[0];
  return fontStacks[lang] || fontStacks.en;
}
```

Inject into CSS:

```css
/* src/styles/fonts.css */
@font-face {
  font-family: 'Inter';
  src: url('/fonts/inter-latin.woff2') format('woff2');
  font-weight: 100 900;
  unicode-range: U+0000-00FF;
  font-display: swap;
}

@font-face {
  font-family: 'Inter';
  src: url('/fonts/inter-arabic.woff2') format('woff2');
  font-weight: 100 900;
  unicode-range: U+0600-06FF, U+0750-077F;
  font-display: swap;
}
```

## Free Tier vs. Scaled Production

| Feature | Free Tier | Scaled |
|---------|-----------|--------|
| Translation API | Manual editing | DeepL / Google Cloud Translation |
| Locale count | 1 (English) | 40+ |
| Caption translation | Manual | Automated with glossary support |
| Asset localization | None | Per-locale image/font pipelines |
| Number formatting | Browser default | CLDR via `react-intl` |
| Date/time | UTC only | Timezone-aware per user |
| Font loading | System fonts only | Web font subsetting by script |
| Region detection | None | GeoIP + cookie override |
| Content delivery | Single origin | CDN with locale-aware routing |
| Storage region | Local | Data sovereignty per region |

## Translation Workflow

### Adding a New Locale

1. Create message file: `src/i18n/messages/<locale>.json`
2. Add to `SUPPORTED_LOCALES` in `src/utils/locale.ts`
3. Add font stack in `src/config/fonts.ts`
4. Run `npm run i18n:extract` to sync keys
5. Upload to Crowdin: `crowdin upload sources`
6. Download translations: `crowdin download`
7. Compile: `npm run i18n:compile`
8. Test with pseudo-locale: `VITE_LOCALE=en-XA npm run dev`

### Translation Review Checklist

- [ ] No hardcoded strings in JSX
- [ ] All plural forms present for target language
- [ ] Text expansion tested (German is ~30% longer than English)
- [ ] Date formats verified against CLDR
- [ ] Currency symbols positioned correctly
- [ ] Icon mirroring applied for RTL locales
- [ ] Screenshots updated for locale-specific content

## Monitoring

Track localization quality with Sentry's i18n context:

```ts
Sentry.setContext('i18n', {
  locale: currentLocale,
  fallbackUsed: wasFallback,
  missingKeys: missingTranslationKeys,
});
```

Alert on missing translation rate > 0.5% per locale.
